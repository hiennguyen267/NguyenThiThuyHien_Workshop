###  [Blog 1 - Optimizing Kubernetes Security with Session Policies in Amazon EKS Pod Identity](3.1-Blog1/)
This blog introduces Amazon EKS Pod Identity with Session Policies, a security enhancement that simplifies AWS IAM permission management for Kubernetes workloads. It explains how Session Policies dynamically restrict permissions for individual pods without requiring numerous IAM roles, helping organizations reduce operational complexity while maintaining the principle of least privilege. The article also presents the architecture, key benefits, and a practical use case in a pharmacy management system running on Amazon EKS.

###  [Blog 2 - Data Masking in Amazon RDS for Oracle](3.2-Blog2/)
This blog summarizes my learning about Data Masking in Amazon RDS for Oracle based on an AWS Database Blog article. It explains why sensitive production data should be protected before being used in development or testing environments, introduces the concept of data masking, and describes an automated workflow using AWS services such as Amazon RDS, EventBridge Scheduler, and AWS Secrets Manager. The article also highlights the importance of protecting sensitive information while maintaining realistic data for application development and testing.

###  [Blog 3 - Understanding Updates to the AWS Well-Architected Framework Guidance](3.3-Blog3/)
This blog summarizes my understanding of the AWS Architecture Blog article “Announcing updates to the AWS Well-Architected Framework guidance.” It introduces the purpose of the AWS Well-Architected Framework, explains its six architectural pillars, and highlights the importance of continuously reviewing cloud workloads instead of treating architecture as a one-time design task. The article also shares key lessons learned and discusses how the framework can be applied to the PharmaCare AI project to improve security, reliability, performance, cost optimization, and operational efficiency.
