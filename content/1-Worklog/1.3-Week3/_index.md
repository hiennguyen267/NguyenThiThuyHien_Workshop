### Week 3 Objectives:

* Understand the fundamentals of Amazon Elastic Compute Cloud (Amazon EC2).
* Learn how to launch, configure, and manage EC2 instances.
* Practice connecting to EC2 instances using SSH and perform basic Linux administration tasks.

* Developed a comprehensive understanding of Amazon Elastic Compute Cloud (Amazon EC2).

* Learned the core components of EC2, including:

  * Amazon Machine Image (AMI)
  * Instance Types
  * Key Pair
  * Security Group
  * Elastic IP Address
  * Amazon EBS (Elastic Block Store)

* Understood the EC2 instance lifecycle, including:

  * Launch
  * Start
  * Stop
  * Reboot
  * Terminate

  * User and permission management
  * Process monitoring
  * Package installation and system updates
  * Basic networking commands

* Understood the relationship between EC2, VPC, Security Groups, and Elastic IP in a cloud environment.

* Gained practical experience in deploying and managing virtual servers on AWS.

* Improved troubleshooting skills by identifying and resolving common SSH connection and security configuration issues.

* Documented the EC2 deployment process and laboratory results for future reference.

* Built a solid foundation for deploying cloud applications and preparing for networking, storage, and database services in the following weeks.
